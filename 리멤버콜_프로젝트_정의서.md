# 리멤버 콜 (Remember Call) — 프로젝트 정의서

> 새 채팅방이나 클로드 코드에 이 문서를 그대로 붙여넣으면 프로젝트 맥락이 이어집니다.
> 이미 확정된 내용이며, 처음부터 다시 설명할 필요가 없습니다.

---

## 1. 한 줄 소개

자녀의 음성을 AI로 합성해(ElevenLabs) 고령의 부모님께 **정기 안부 전화 + 대화형 인지 퀴즈**를 제공하고, 통화 발화 데이터를 분석해 자녀에게 인지 건강 리포트를 주는 **Voice-Only 시니어 케어 서비스**.

- 부모님(사용자): 앱 설치·가입 없이 평소처럼 걸려온 전화를 받기만 함
- 자녀(구매자, 3040): 모바일 앱으로 음성 등록·스케줄 관리·리포트 확인
- 비즈니스 모델: B2C 정기 구독 (향후 B2G/B2B 확장)
- 브랜드 태그라인: **"기억을 지키는 가장 다정한 연결"** (고정)

---

## 2. 만들고 있는 산출물 (2개)

### A. 소개 홈페이지 (웹)
- 목적: 자녀(구매자)에게 서비스를 소개하고 신청을 받는 마케팅 랜딩 사이트
- 구성: 메인 랜딩(1페이지 스크롤) + 신청 페이지 + 약관 페이지
- 현재 상태: `index.html`, `terms.html` 제작 완료

### B. 자녀용 앱 (모바일 앱 프로토타입)
- 목적: 자녀가 리포트를 받고, 음성·히스토리·스케줄을 관리하는 앱
- 현재 상태: 진입화면·온보딩·홈·리포트·추세·위험알림 화면 전환 프로토타입 제작 완료 (`remembercall-app/`)

### 폰트
- 영문 제목/워드마크: `Cormorant Garamond` (세리프)
- 한글 제목: `Noto Serif KR` (세리프) — 다정·서정적 무드
- 본문: `Pretendard` (산세리프) — 가독성

### 톤앤매너
- 따뜻하고 정제된(웜 에디토리얼) 분위기, 넓은 여백, 부드러운 모션
- 홈페이지: 풀스크린 히어로(가족 사진/영상 위 흰 세리프 글씨) → 스크롤 스토리텔링
- 앱: 멀티캠퍼스류 피드형 구조 + 카드 UI + 하단 탭바(홈·리포트·추세·내 정보)
- **금지: "치매·검사·측정·점수" 등 평가성 단어** (부모님 낙인 효과 방지). 자녀 앱 내부 표현은 가능하나 부모님 통화/화면에는 절대 금지

### 로고
- 사용자가 직접 제작한 파일 사용 (AI가 임의로 다시 그리지 않음)
- 모티프: 부모·자녀 옆모습이 하트를 이룸 + 음성 파형 + 'Remember Call' 세리프 워드마크, 골드 톤
- 배경 투명 처리 완료: `logo.png`(원본 비율), `logo-mark.png`(여백 잘라낸 버전)
- 진입화면: 어두운 배경에 전체 로고 / 헤더: 심볼+글자 작게

## 4. 핵심 화면 구조

### 홈페이지 (스크롤 순서)
헤더 → 히어로 → 작동 원리(4단계) → 핵심 기능(3종) → 안심·보안(3종) → 요금제(월/연) → FAQ → 푸터 / + 신청 페이지 + 약관 페이지

### 자녀용 앱
1. 진입화면(스플래시) — 로고 + 태그라인
2. 온보딩 1·2·3 — 안부 전화 / 인지 체크 / 알림 안내 (직접 그린 SVG 일러스트)
3. 홈(피드형) — 케어 저니, 오늘 통화, 인지 현황 요약, 위험 알림 배너
4. 리포트 상세 — 4지표(답변 지연·망설임 빈도·정답률·기억 망각 카테고리) + 공유
5. 추세 분석 — 주간/월간 그래프, 전월 대비 임계치(+30%) 경고
6. 위험 알림(Risk Alert) — 감지 근거 + 정밀 검진 권장 + 병원 찾기/공유
7. (설정/입력) 음성 녹음, 가족 히스토리 관리, 통화 스케줄 설정, 계정·구독·알림

## 5. 리포트 분석 지표 (앱 핵심 기능)

- **답변 지연 시간:** 질문 종료~첫 발화까지 걸린 시간
- **망설임 빈도:** "어…", "그게…" 등 단어 인출 실패로 인한 침묵·공백 횟수
- **오답 누적 흐름:** 주간/월간 오답률 추세 변화
- **특정 기억 망각률:** 카테고리별(인명·지명 등) 못 떠올리는 단어 군집
- **위험 알림(High-Risk Push):** 인명/지명 연속 오답 또는 평균 답변 지연이 전월 대비 30% 초과 시, 경도인지장애 가능성 경고를 자녀에게 자동 발송

## 6. 데이터 항목 (참고)

- 계정: 자녀 정보, 부모님 프로필, 구독·결제(카드 정보는 PG 토큰만 저장)
- 음성: 녹음 원본(생성 즉시 영구 삭제), AI 음성 모델 ID(외부 서버, DB엔 매핑 ID만)
- 퀴즈 원천: 가족 히스토리(출생지·직장·태명·기념일), LLM 생성 퀴즈 스크립트
- 통화: 스케줄, 세션 로그, 부모님 답변(STT 텍스트)
- 분석: 정답/오답, 답변 지연, 망설임, 추세·망각 통계, 위험 알림 이력
- 홈페이지에서 실제 다루는 데이터: 신청 폼(이름·연락처·이메일·동의), 문의

## 7. 제약 조건 (중요)

### 기술적
- Claude Code로 개발, 코딩 지식 없이 구현 가능한 범위
- 사용 가능: HTML/CSS/JS 정적 사이트, 무료 폰트/아이콘/이미지, 외부 무료 폼(Formspree·Tally), 무료 배포(Vercel·Netlify)
- 사용 불가(소개·목업으로만 표현): 실제 음성 복제·자동 전화 발신·STT/LLM 퀴즈·실시간 분석, 로그인·실시간 DB·결제(PG)
- **즉, 홈페이지와 앱은 "실제 제품"이 아니라 디자인·흐름을 보여주는 정적 사이트/프로토타입**

### 비즈니스
- 개발 기간: 약 7~10일 (코드는 Claude Code가 작성, 콘텐츠·검수 중심)
- 예산: 0원 ~ 약 2~3만원(도메인 정도)
- 혼자 운영 가능한 규모

## 8. 성공 지표 (요약)

- 홈페이지: 사전신청 전환율(North Star), 핵심 섹션(안심·보안) 도달률, CTA 클릭률
- 서비스: 활성 구독 가구 수(North Star), 온보딩 완료율, 통화·퀴즈 완료율, 구독 유지율, 자녀 대시보드 열람률, NPS

## 9. 작업 시 주의사항 (AI에게)

- 로고는 사용자 제작 파일(`logo.png`/`logo-mark.png`)을 그대로 사용. 임의로 다시 그리지 말 것
- 베이지 팔레트·세리프 제목·태그라인 고정
- 부모님 대상 문구에 평가성 단어(치매·검사·측정·점수) 금지
- 홈페이지/앱은 정적 프로토타입 범위. 서버 기능은 구현 대상 아님
